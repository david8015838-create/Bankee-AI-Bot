📝 新增 SOP 指南

有新的流程要上傳？跟著步驟做，讓 AI 助理學會新知識

  • 1

    收集原始資料

    把手上的資料整理好，不需要先整理格式。

    Word / LINE 截圖 / 筆記 / 口頭說明打字都可以
  • 2

    用 AI 整理成 Markdown 格式

    切換到「AI 整理 Prompt」頁，複製 Prompt，貼到 ChatGPT / Claude / Gemini，把原始資料貼在後面送出。AI 會先把散落各處的同一主題內容合併，再寫成 Markdown——不用擔心原始資料格式凌亂。

  • 3

    自己對照檢查

    切換到「上傳前檢查」頁，逐項確認，全部通過才繼續。

  • 4

    存成 .md 或 .txt 檔

    把 AI 輸出的文字複製，用記事本或任何文字編輯器貼上，儲存為 流程名稱.md

  • 5

    上傳到 Google Drive

    點下方連結進入資料夾 → 點「＋新增」→「檔案上傳」→ 選擇剛儲存的檔案。

    📂 開啟 Bankee AI-Bot 資料夾
  • 6

    重新整理網頁驗證

    按 Cmd+R（Mac）或 F5（Windows）重新整理，用口語問一個新 SOP 應該能回答的問題，確認右側顯示剛上傳的文件名稱。

RAG 優化 Prompt（三階段）

複製以下 Prompt，貼到任何 AI 工具（ChatGPT / Claude / Gemini），再把你的原始資料貼在最後面。
三階段流程：① 理解與統整原始資料 → ② 撰寫 RAG 優化 Markdown → ③ AI 自我檢查

AI 整理完後，逐項確認，全部通過才能上傳到 Google Drive：

文件骨架
  • ☐有「一句話說明」（一句話概括文件內容）
  • ☐有「適用對象」和「不適用對象」
  • ☐有「流程步驟」主體內容
  • ☐有「常見問題」區塊
  • ☐有「最後更新」（日期 + 負責人）
步驟完整性
  • ☐每個步驟都有：操作者、系統全名、具體動作、完成標誌
  • ☐條件分支每條路徑都完整寫出（沒有「依情況而定」）
  • ☐系統名稱每次都寫全名，沒有用「它」「該系統」代替
  • ☐⚠️ 警告放在對應步驟正下方
常見問題
  • ☐至少 8 個問答
  • ☐涵蓋 6 角度：怎麼開始、找誰、截止時間、出錯怎辦、特殊情況、結果確認
  • ☐每個答案完整獨立（不依賴其他段落、不說「同上」）
  • ☐同一個問題有 2～3 種不同問法
  • ☐重要關鍵詞同時出現在步驟和問答中
安全與格式
  • ☐沒有「如上所述」「同前」「參見步驟 X」等交互引用
  • ☐不確定的地方有標記 [待確認]，沒有 AI 自己編造的資料
  • ☐純文字，沒有圖片（圖片內容已轉為文字描述）
  • ☐文件長度 800～2000 字（過長請拆分）

🔄 多開 Key 延長免費額度

  1. 1每個 Google 帳號可以建立多個 GCP 專案，每個專案各自有獨立的免費額度
  2. 2重複上方步驟，在「Create API key in new project」時系統會自動建立新專案（例如 My Project 2、My Project 3）
  3. 3也可以用不同的 Google 帳號各自申請，再把所有 Key 都加進來
  4. 4這個助理會自動輪流使用，一組 Key 達到上限就換下一組

加 3 組 Key 就等於每天 4,500 次，一般日常使用綽綽有餘。
