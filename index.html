📌 一句話總結
這是一個純瀏覽器運作的 SOP 智慧助理，使用語意向量搜尋技術從你的 SOP 文件中找出與問題相關的內容，再交由 AI 模型生成自然語言回答。

1 整體架構

這個系統沒有後端伺服器，所有資料和運算都發生在你的瀏覽器內。

你的瀏覽器
├── localStorage（個人筆記、設定、API Key、向量索引）
├── Google Drive API（讀取/寫入 SOP 文件）
└── AI API（Gemini / OpenAI / Anthropic）
    ├── Embedding API（把文字轉成向量）
    └── Chat API（生成回答）

所有 API 都是從你的瀏覽器直接打給服務商，沒有任何中間伺服器收集你的問題或文件內容。

2 文件如何載入

  1. 點「📁 開啟文件庫」後，系統會去指定的 Google Drive 資料夾抓取所有文件清單。
  2. 每份文件的內容（Markdown 或純文字）會被下載到瀏覽器，並存在本機快取（24 小時內不再重新下載）。
  3. 下載完成後，系統會立刻在背景開始建立語意索引（見下一段）。
  4. 狀態列會即時顯示進度，例如「建立語意索引 3/12...」。

3 核心技術：Chunk-level Embedding

這是整個系統最關鍵的部分。Embedding 的意思是把一段文字翻譯成一組數字陣列（向量），意思相近的文字會有相近的向量。

📊 範例
「請假要找誰審核」  → [0.12, -0.87, 0.43, ...]
「假單簽核流程」    → [0.11, -0.85, 0.41, ...]  ← 很相近
「報帳單據規定」    → [0.67, 0.23, -0.12, ...]  ← 差很遠

「Chunk-level」表示我們不是把整份文件壓成一個向量，而是按 Markdown 的標題（## 和 ###）切成小段，每段各自 embedding。

為什麼要切小段？

想像你有一份「工作基本資訊」文件，裡面有 20 條不同性質的資訊（VPN、印表機、資料夾路徑、HR 系統...）。

❌ 若整份文件只算一個向量 → 向量會是「雜項資訊彙整」這個模糊的語意，使用者問「財務報表在哪」時很難命中。

✅ 切成 chunks 後 → 每條資訊各自有自己的向量，「財務報表：\\server\finance\reports」這段會直接命中。

所有 chunk 的向量會存在你瀏覽器的 localStorage 裡，下次開啟不用重算。

4 提問時發生什麼事

完整流程
  1. 你輸入問題 → 送給 Gemini Embedding API → 拿到問題的向量
  2. 把問題向量 vs 所有 chunk 向量做餘弦相似度比對（cosine similarity）
  3. 每份文件取最高相似度分數，依分數排名
  4. 套用三道護欄：相似度 ≥ 0.45、最多 5 份、總字數不超過 60,000 字
  5. 把命中的整份文件（不只 chunk）注入給 Chat AI 當參考資料
  6. AI 根據文件回答問題，並標示來源
💡 為什麼用 chunk 找，卻注入整份文件？

Chunk 用來精準定位「答案藏在哪份文件」，但 AI 需要文件的完整上下文才能準確回答。所以搜尋粒度細，注入粒度粗，兩者兼顧。

5 📌 常駐文件功能

有時候某份文件是「不管問什麼都應該參考」的（例如部門通訊錄、系統路徑速查表），這時可以點文件名稱旁的 📌 把它設為常駐。

常駐文件 vs 一般文件
  • 一般文件：要被語意搜尋命中才會注入
  • 常駐文件（📌）：每次提問都必定注入，繞過搜尋

適合用在你常常需要查路徑、聯絡人、系統設定的速查型文件。

7 三道護欄：防止注入太多內容

為了避免文件量變多時，把太多內容塞給 AI 導致品質下降或費用爆炸，系統設置三層保險：

  1. 相似度閾值（0.45）：低於此分數的文件直接捨棄，避免硬塞不相關的內容
  2. Top N 限制（最多 5 份）：不管多少份文件超過閾值，最多只取前 5 份
  3. 總字數上限（60,000 字）：累計超過就停止加入，預留 AI 思考的 context 空間

8 快取與失效機制

  • 文件內容快取：24 小時內不會重新從 Drive 下載
  • 向量快取：每個 chunk 帶有內容簽章，內容沒變就不重新 embedding
  • 編輯文件後：簽章改變，會自動觸發該文件重新 embedding
  • 從 Drive 刪除的文件：下次載入時其向量會被自動清除（孤兒清理）
  • 強制重建：點「🔄 重新整理文件」會清除所有快取重新載入

9 若語意搜尋失敗會怎樣

以下情況系統會自動退回成「使用所有勾選的文件」的舊行為，確保功能不中斷：

  • 使用 OpenAI 或 Anthropic 作為主要 AI 時（embedding 目前只支援 Gemini）
  • 尚未建立任何索引時
  • Embedding API 暫時失敗時
  • 沒有任何文件相似度超過 0.45 時

10 資料流向與隱私

會送出瀏覽器的東西
  • 送到 Google Drive：你的 OAuth token，用於下載 / 上傳文件
  • 送到 Gemini Embedding API：文件內容 chunks（建立索引時）、你的提問
  • 送到 Chat API：系統提示 + 命中的文件全文 + 你的問題
不會離開瀏覽器的東西
  • 你的 API Keys（只存 localStorage）
  • 個人筆記、待辦事項、通訊錄、常用連結
  • 向量索引快取
  • 對話歷史（重新整理頁面就消失）

11 使用的模型

  • Embedding 模型：Gemini text-embedding-004（768 維向量，目前免費）
  • Chat 模型：依設定，預設 Gemini 2.5 Flash Lite。可在「⚙️ API 設定」中切換
